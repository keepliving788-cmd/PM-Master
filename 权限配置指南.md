# 🔒 飞书应用权限配置指南

## ❌ 当前问题

错误信息：`permission denied: tenant needs read permission`

**原因：** 你的飞书应用缺少知识库读取权限，或权限未生效。

---

## ✅ 完整解决步骤

### 第1步：登录飞书开放平台

访问：https://open.feishu.cn/

使用管理员账号登录。

---

### 第2步：找到你的应用

1. 在控制台找到应用：`cli_a91379879838dcee`
2. 点击进入应用详情

---

### 第3步：配置权限（重要！）

#### 3.1 进入权限管理

- 点击左侧菜单：**"权限管理"** 或 **"Permissions & Scopes"**

#### 3.2 添加必需权限

搜索并**开通**以下权限：

| 权限名称 | 权限代码 | 说明 | 状态 |
|---------|---------|------|------|
| 获取知识空间信息 | `wiki:wiki:readonly` | **必需** | ⚠️ 需开通 |
| 查看、下载和导出多维表格 | `bitable:app:readonly` | 可选 | |
| 获取文档内容 | `docx:document:readonly` | **必需** | ⚠️ 需开通 |
| 获取云文档信息 | `drive:drive:readonly` | 推荐 | |

**关键权限：**
```
✅ wiki:wiki:readonly     ← 必须开通！
✅ docx:document:readonly ← 必须开通！
```

#### 3.3 确认权限已开通

截图示例应该显示：
```
✓ wiki:wiki:readonly          已开通
✓ docx:document:readonly      已开通
```

---

### 第4步：发布应用（关键！）

**非常重要：添加权限后必须重新发布应用！**

1. 点击右上角 **"版本管理与发布"** 或 **"Publish"**
2. 点击 **"创建版本"**
3. 填写更新说明：如 "添加知识库读取权限"
4. 点击 **"保存并提交审核"**（或直接发布，取决于你的企业设置）

**发布状态：**
- ✅ 已发布 → 可以使用
- ⏸️ 待审核 → 需等待审核通过
- ❌ 未发布 → 权限不会生效

---

### 第5步：等待权限生效（1-2分钟）

权限配置后需要短暂等待才会生效：

```bash
# 等待1-2分钟后测试
sleep 120
python3 test_api_access.py
```

## ❓ 常见问题

### Q1: 权限已开通但仍然报错？

**解决方案：**
1. 确认应用已重新发布
2. 等待5分钟让权限完全生效
3. 清除缓存重试：
   ```bash
   rm -rf ~/.cache/
   python3 test_api_access.py
   ```

### Q2: 找不到"权限管理"菜单？

**可能原因：**
- 你的账号不是应用管理员
- 需要联系创建该应用的管理员

**解决方案：**
联系飞书管理员，请求管理员权限或请管理员帮你配置。

### Q3: 应用审核中，需要等多久？

**时间：**
- 小型企业：立即生效或几分钟
- 大型企业：可能需要管理员审核，1-24小时

**临时方案：**
如果等不及，可以：
1. 创建一个新的测试应用
2. 配置相同权限
3. 立即发布使用

### Q4: 我不是管理员，怎么办？

**步骤：**
1. 联系你们公司的飞书管理员
2. 提供应用ID：`cli_a91379879838dcee`
3. 请求开通以下权限：
   - `wiki:wiki:readonly`
   - `docx:document:readonly`
4. 请求管理员发布应用
