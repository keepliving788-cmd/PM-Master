## 📝 配置步骤（5分钟）

### 第1步：登录飞书开放平台
访问：https://open.feishu.cn/app

### 第2步：找到你的应用
1. 在应用列表中搜索：`cli_a91379879838dcee`
2. 或查找应用名称
3. 点击进入应用详情页

### 第3步：配置权限
1. 点击左侧菜单 **"权限管理"** 或 **"Permissions & Scopes"**

2. 在搜索框搜索并添加以下权限：

   **权限1：wiki:wiki:readonly**
   ```
   搜索: wiki
   找到: "获取知识空间信息"
   权限代码: wiki:wiki:readonly
   点击: "申请权限" 或 "开通"
   ```

   **权限2：docx:document:readonly**
   ```
   搜索: docx 或 document
   找到: "获取文档内容" 或 "查看、下载和导出云文档"
   权限代码: docx:document:readonly
   点击: "申请权限" 或 "开通"
   ```

3. 确认两个权限都显示为 **"已开通"** 或 **"已授权"**

### 第4步：重新发布应用（关键！）
⚠️ **这一步最重要！不发布则权限不会生效！**

1. 点击顶部或侧边的 **"版本管理与发布"**
2. 点击 **"创建版本"**
3. 填写版本说明，例如：
   ```
   添加知识库读取权限
   - wiki:wiki:readonly
   - docx:document:readonly
   ```
4. 点击 **"保存"** → **"提交审核"** 或 **"发布"**
5. 确认应用状态变为 **"已发布"** 或 **"可用"**

### 第5步：等待生效
- **小企业**：1-2分钟
- **中型企业**：2-5分钟
- **大型企业**：可能需要管理员审核（5分钟-24小时）

## ❓ 常见问题

### Q1: 找不到这两个权限？
**搜索关键词：**
- `wiki` → 找到 "获取知识空间信息"
- `docx` 或 `document` → 找到 "获取文档内容"
- `云文档` → 可能也能找到相关权限

### Q2: 权限已开通但仍报错？
**可能原因：**
1. 未重新发布应用（最常见）
2. 权限还在生效中（等待5分钟）
3. 缓存问题（清除后重试）

**解决方案：**
```bash
# 确认已重新发布应用
# 等待更长时间
sleep 300
python3 test_api_access.py

# 清除缓存
rm -rf ~/.cache/lark_oapi
python3 test_api_access.py
```

### Q3: 权限申请需要审核？
**情况：**
- 大型企业可能需要管理员审核
- 审核时间：几分钟到24小时不等

**临时方案：**
- 联系飞书管理员加速审核
- 或创建一个新的测试应用（可能秒过）

### Q4: 不是应用管理员怎么办？
**解决方案：**
联系应用创建者或企业管理员：
1. 提供应用ID：`cli_a91379879838dcee`
2. 请求开通权限：
   - `wiki:wiki:readonly`
   - `docx:document:readonly`
3. 请求重新发布应用
