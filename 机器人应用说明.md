## ⚠️ 关键差异

### 机器人应用 vs 普通应用

| 特性 | 机器人应用 | 普通应用/网页应用 |
|------|-----------|------------------|
| **主要用途** | 对话、消息、卡片 | API调用、数据访问 |
| **权限类型** | 机器人权限 | 应用权限 |
| **知识库访问** | ⚠️ 可能受限 | ✅ 完全支持 |
| **鉴权方式** | tenant_access_token | tenant_access_token |

---

## 🔍 机器人能否访问知识库？

### ✅ 理论上可以

机器人应用**可以**访问知识库，但需要：

1. **添加对应权限**
   - `wiki:wiki:readonly`
   - `docx:document:readonly`

2. **使用正确的token**
   - tenant_access_token（应用级）✅ 我们在用
   - user_access_token（用户级）

3. **机器人可能需要额外配置**
   - 有些企业限制机器人访问某些资源
   - 需要管理员授权

---

## 🎯 两种解决方案

### 方案A：继续使用机器人应用（推荐先试试）

**操作步骤：**

1. 确认机器人应用已添加权限：
   - `wiki:wiki:readonly`
   - `docx:document:readonly`

2. 重新发布机器人

3. 测试是否能访问：
   ```bash
   python3 test_api_access.py
   ```

**可能的问题：**
- 某些企业可能限制机器人访问知识库
- 权限配置可能和普通应用不同

---

### 方案B：创建专用应用（更稳妥）

如果机器人无法访问知识库，创建一个新的**网页应用**：

**步骤：**

1. 在飞书开放平台创建新应用
   - 应用类型：**网页应用** 或 **自建应用**
   - 不要选择"机器人"

2. 配置权限：
   - `wiki:wiki:readonly`
   - `docx:document:readonly`

3. 发布应用

4. 更新 `.env` 文件：
   ```bash
   FEISHU_APP_ID=新应用的ID
   FEISHU_APP_SECRET=新应用的SECRET
   ```

5. 测试：
   ```bash
   python3 test_api_access.py
   ```
